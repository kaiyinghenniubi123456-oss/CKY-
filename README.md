# 教学全流程 Skills

WorkBuddy (CodeBuddy) Skills 合集 —— 由（CC）维护，面向一线教师的 AI 教学工作流。

从拿到一本教材到走进教室上课，全程 6 个 Skill 覆盖：**教材解读 → 资源检索 → 教案填写 → PPT 制作 → 视频下载**，外加一个编排层把整条链串起来。

---

## 这是什么

把"备课"这件事拆成 5 个可独立使用、又可联动编排的 AI 技能模块。每个模块单独拿出来就能用；装上编排层后，一条指令跑完全流程。

### 核心设计：编号联动系统

整个技能包的灵魂是一条**编号系统**，贯穿所有模块：

```
重点1-5  ←──  教材解读产出
难点1-12 ←──  教材解读产出
资源#1-#15 ←── 资源检索产出（映射到重点/难点）
```

- **单独用**：编号系统不起作用，各模块独立干活，互不影响
- **串起来用**：编号自动传递，下游模块严格引用上游产出，PPT 每页标注对应重点+难点+资源，教案每个环节标注知识点来源

### 一例到底原则

全程用**同一个案例**的不同侧面串讲。比如用「锅圈食汇」：导入用品牌故事，4Ps用其渠道策略，营销道德用其3·15案例，创新用其数字化转型——学生不因频繁切换案例而分散注意力。

## Skill 目录

| # | Skill 名称 | 功能 | 可否单独使用 |
|---|-----------|------|:---:|
| 0 | **teaching-pipeline** | 编排层：流程控制 + 编号传递 + 质量检查（C1-C4） | 可独立 |
| 1 | **textbook-interpretation** | 教材深度解读：提炼重点、预判难点、生成突破图和资源需求单 | 可独立 |
| 2 | **textbook-resource-workflow** | 资源联动检索：按需求单三阶段分组搜索，输出可验证的资源链 | 可独立 |
| 3 | **teaching-plan-custom** | 教案填写·通用模板版：扫描任意学校模板结构，按字段精准填写 | 可独立 |
| 4 | **teaching-ppt-swiss** | 教学 PPT 一站式设计：课堂版/比赛版，逐页标注重点+难点+资源 | 可独立 |
| 5 | **z-video-downloader** | 视频下载：YouTube/B站/抖音多平台支持，含降级策略 | 可独立 |

> 每个 Skill 文件夹内均有独立的 `SKILL.md`（技能定义文件），WorkBuddy 读取该文件来决定何时触发、如何执行。

## 安装方式

### 方式一：安装全部（推荐）

1. 下载 `教学全流程skills.zip`
2. 解压后得到 6 个文件夹
3. 将文件夹复制到 `~/.workbuddy/skills/` 目录
4. 完成 — 在 WorkBuddy 对话中直接说需求即可触发

### 方式二：只装需要的模块

比如只想做 PPT，那就只装 `teaching-ppt-swiss` 一个文件夹到 `~/.workbuddy/skills/`。给它一份教材解读报告 + 资源检索报告就能生成 PPT。

### 安装路径说明

| 平台 | 路径 |
|------|------|
| WorkBuddy (macOS) | `~/.workbuddy/skills/` |
| WorkBuddy (Windows) | `%USERPROFILE%\.workbuddy\skills\` |

## 使用方式

### 场景 A：跑全流程

> 对 WorkBuddy 说："帮我备课，这是教材内容：[贴入教材文本]"

触发 `teaching-pipeline`（编排层），自动按 5 步顺序执行：

```
Step 1  教材解读    →  产出 5重点 12难点 + 资源需求单
Step 2  资源检索    →  产出 15条资源（认知5 + 案例8 + 实训2）
Step 3  PPT制作 ┐
              ├── 可并行
Step 4  教案填写 ┘
Step 5  视频下载    →  下载课堂用视频到本地
```

每步完成后输出衔接信息，用户确认后进入下一步。可随时说"暂停"中断，说"继续"恢复。

### 场景 B：单独用某个模块

| 需求 | 说什么 | 触发 Skill |
|------|--------|-----------|
| 解读教材 | "帮我解读这段教材" | textbook-interpretation |
| 找教学资源 | "帮我找教学资源" | textbook-resource-workflow |
| 填教案 | "帮我按这个模板填教案" | teaching-plan-custom |
| 做 PPT | "帮我做个教学PPT" | teaching-ppt-swiss |
| 下视频 | "帮我下载这个视频" | z-video-downloader |

## 联动协议详解

### 编号传递规则

```
教材解读产出：重点1 / 重点2 / ... / 重点5
              难点1 / 难点2 / ... / 难点12

资源检索产出：资源#1（→重点1·难点2）
              资源#2（→重点1·难点1）
              ...

PPT 每页标注：知识点：重点N | 难点：难点N | 资源：资源#N
教案每环节标注：对应重点N / 难点N / 资源#N
```

### 质量检查点（编排层执行）

| 检查点 | 检查内容 | 标准 |
|--------|---------|------|
| C1 | 解读覆盖率 | 重点≥3个、难点≥8个 |
| C2 | 资源覆盖率 | 每个重点至少1条资源 |
| C3 | PPT联动率 | PPT页面100%标注重点+难点+资源 |
| C4 | 教案联动率 | 教案每个环节标注编号来源 |

### PPT 五条强绑定约束

1. 每页内容必须来源于教材解读报告，禁止引入解读外知识点
2. 每页必须嵌入对应资源并标注来源（资源编号+名称+链接）
3. 每页必须标注对应知识点编号和难点编号
4. PPT 结构必须与解读报告重点/难点对齐
5. 全部资源必须嵌入，无遗漏无额外
